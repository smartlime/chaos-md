# Тест 03: Отказ диска (disk fail)

## Справка (`./03-disk-fail.sh -h`)

Только **`-1`**. Ключи: `-d` устройство, `-t`, `-H`, `-r` нет (рестарт по умолчанию), **`--no-restart`**, `-C`, `-D`. **Не включён** в `run-all.sh` (ручной контроль).

```bash
./03-disk-fail.sh -1 -t 600 -d vdb
./03-disk-fail.sh -1 -t 600 --no-restart
./03-disk-fail.sh -D
```

## Что делает тест

Меняет **GPT partition name** (то, что видно как partlabel в `/dev/disk/by-partlabel/`) для выбранной партиции на диске. YDB обычно ищет диски по метке; метка «хаоса» (`CHAOS_DISK_LABEL_CHAOS`, по умолчанию `test`) имитирует потерю нужного диска без удаления блочного устройства из ядра.

После окна `-t` фоновый таймер на хосте возвращает штатную метку (`CHAOS_DISK_LABEL_NORMAL`, по умолчанию `ydb_disk_ssd_01`), выполняет `partx -u`. По умолчанию затем выполняется **`systemctl restart`** сервиса storage (см. `nemesis/proc.sh`), чтобы кластер подхватил диск.

Подготовка хостов: пакет **`gdisk`** (`sgdisk`) ставится скриптом **`prepare-hosts.sh`**.

## Команды (ручная проверка на ноде)

### «Убрать» диск для YDB (сменить метку партиции 1)

```bash
sudo sgdisk -c 1:"test" /dev/vdb
sudo partx -u /dev/vdb
```

### Проверить метки

```bash
ls -l /dev/disk/by-partlabel/
lsblk
```

### Вернуть диск

```bash
sudo sgdisk -c 1:"ydb_disk_ssd_01" /dev/vdb
sudo partx -u /dev/vdb
```

При необходимости перезапустите storage (как в автоматическом сценарии).

### Протокол: диск INACTIVE в dstool

Если после возврата метки `ydb-dstool pdisk list` показывает диск как **INACTIVE**, его иногда активируют вручную (`pdisk set --status ACTIVE`) и перезапускают pdisk/storage. В автоматизации заложены смена метки, `partx` и рестарт storage; при специфичных ответах кластера может понадобиться dstool вне скрипта.

## Механика скрипта

1. Проверка `-C`: `lsblk`, симлинки в `/dev/disk/by-partlabel/` для хаос- и рабочей метки.
2. Применение: `sgdisk -c`, `partx -u`, фоновый таймер на возврат метки через `-t` с.
3. Ожидание окна локально; после таймера — опционально рестарт storage (`--no-restart` отключает).
4. `-D`: снять таймер, сразу вернуть метку и `partx -u`, затем при необходимости рестарт storage.

## Параметры в `env.sh`

| Переменная | По умолчанию | Описание |
|------------|--------------|----------|
| `DEFAULT_DISK_DEVICE` | `vdb` | Имя устройства (`vdb` или `/dev/vdb`) |
| `CHAOS_DISK_PARTNUM` | `1` | Номер партиции для `sgdisk -c` |
| `CHAOS_DISK_LABEL_CHAOS` | `test` | Метка во время теста |
| `CHAOS_DISK_LABEL_NORMAL` | `ydb_disk_ssd_01` | Рабочая метка |
| `CHAOS_DISK_RESTART_STORAGE` | `true` | Рестарт storage после восстановления |

## Что наблюдать

- Исчезновение нужной метки в `by-partlabel` на время окна.
- Реакцию YDB на «пропажу» диска по метке и сходимость после восстановления и рестарта storage.
