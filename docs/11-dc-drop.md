# Тест 11: блокировка YDB-интерконнекта на всём ДЦ

## Справка (`./11-dc-drop.sh -h`)

Только **`-4`**. Порты из **`YDB_PORTS`**. По умолчанию **REJECT**; **`--drop`** — тихий DROP. Ключи: `-t`, `-C`, `-D`.

```bash
./11-dc-drop.sh -4 -t 600
./11-dc-drop.sh -4 --drop -t 600
./11-dc-drop.sh -C
./11-dc-drop.sh -D
```

Семантика правил iptables — как в [тесте 06](06-net-drop.md) (`nemesis/iptables.sh`), но параллельно на всех хостах **`DC_HOSTS`**.

## Отличие от теста 06

| Аспект | Тест 06 | Тест 11 |
|--------|---------|---------|
| Область | Одна нода (`-1`) | Весь ДЦ (`-4`, `DC_HOSTS`) |
| Эффект | Одна нода изолирована | Полная потеря интерконнекта в ДЦ |
| REJECT / DROP | да, `--drop` | да, `--drop` |

Длительность `-t` для обоих тестов — **`DEFAULT_CHAOS_TIMEOUT`** из `env.sh` (обычно 1200 с), если не переопределить.

## Применение

На **каждом** хосте ДЦ параллельно: цепочка `CHAOS_IPTABLES_CHAIN`, четыре правила на порт × iface × стек (IPv4/IPv6). Подробности и пример — в [06-net-drop.md](06-net-drop.md).

## Снятие

На каждом хосте: `iptables -F` / `ip6tables -F` цепочки. Скрипт делает это через `chaos_run_window` по истечении `-t` или сразу по `-D`.

```bash
./11-dc-drop.sh -D
```

## Проверка

```bash
./11-dc-drop.sh -C                              # все хосты DC_HOSTS
./11-dc-drop.sh -C node.example.net           # один хост
```

## Механика

Как тест 06: **нет** фонового nohup на нодах — управляющая машина ждёт `-t` и вызывает teardown. На каждом хосте ДЦ свой независимый набор правил; снятие по `-D` — параллельно на всех.

## Параметры

| Параметр | Ключ | По умолчанию |
|----------|------|--------------|
| Длительность фазы, с | `-t` | `DEFAULT_CHAOS_TIMEOUT` |
| Режим DROP | `--drop` | выкл. (REJECT) |
| Проверка | `-C` [HOST] | — |
| Снять | `-D` | — |

## Что наблюдать

- Полная потеря интерконнекта в ДЦ — жёстче, чем изоляция одной ноды (тест 06)
- Потеря кворума / split-brain в YDB
- Массовые ошибки запросов
- Поведение при одновременном восстановлении связи на всех нодах
- Корректность данных после снятия блокировки
