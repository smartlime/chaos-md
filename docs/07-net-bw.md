# 07-net-bw — Ограничение пропускной способности сети

## Справка (`./07-net-bw.sh -h`)

**`-1`** или **`-4`**. TBF на **весь** исходящий трафик интерфейса (не только YDB-порты). Ключи: `-r` Мбит/с, `-b` burst, `-t`, `-D`.

```bash
./07-net-bw.sh -1 -r 1 -t 600
./07-net-bw.sh -4 -r 500 -t 600
./07-net-bw.sh -D
```

## Назначение

Ограничение исходящей полосы задаётся дисциплиной **TBF (Token Bucket Filter)** на корне qdisc интерфейса: действует на весь исходящий трафик (IPv4 и IPv6), без отбора по портам.

## Как работает TBF

TBF использует модель «токенов»: токены накапливаются в ведре (`burst`) со скоростью `rate`.
Пакет может быть отправлен, только если в ведре достаточно токенов. При превышении скорости
пакеты встают в очередь до `latency`мс, после чего отбрасываются.

```
  tokens накапливаются → ведро (burst) → пакеты уходят со скоростью ≤ rate
                                        → если ведро пусто — ждать или дропать
```

## Параметры TBF

| Параметр   | Описание                                                        |
|------------|-----------------------------------------------------------------|
| `rate`     | Максимальная пропускная способность (напр. `1mbit`)            |
| `burst`    | Размер ведра в байтах — максимальный мгновенный всплеск        |
| `latency`  | Максимальное время ожидания пакета в очереди до отброса        |

## Расчёт burst

**Минимальный burst** ограничен аппаратным таймером ядра (HZ):

```
burst_min = rate_bytes_per_sec / HZ
```

На типичных Linux-серверах `HZ=250`, значит для 1 мбит/с:
```
burst_min = (1_000_000 / 8) / 250 = 500 байт
```

Это технический минимум. На практике такой burst приведёт к очень «дёрганой» передаче
(каждые 4мс — маленький всплеск). Рекомендуемый burst — **1/8 секунды трафика**:

```
burst = rate_bytes_per_sec / 8 = rate_mbit * 1_000_000 / 8 / 8 = rate_mbit * 15625
```

Примеры:

| Rate    | burst (авто)       | Комментарий                     |
|---------|--------------------|---------------------------------|
| 1 мбит  | 15 625 б ≈ 15k    | ~1 MTU (нормальная работа)      |
| 10 мбит | 156 250 б ≈ 156k  |                                 |
| 100 мбит| 1 562 500 б ≈ 1.5M|                                 |

Скрипт вычисляет burst автоматически, минимум 1600 байт. Переопределить: `-b BYTES`.

## Применение ограничения

```bash
# Удалить существующую root qdisc (если есть)
sudo tc qdisc del dev eth0 root

# Ограничить до 1 мбит, burst 15625 байт, очередь до 50мс
sudo tc qdisc add dev eth0 root tbf rate 1mbit burst 15625 latency 50ms
```

## Снятие ограничения

```bash
sudo tc qdisc del dev eth0 root
```

## Проверка

```bash
tc qdisc show dev eth0
# При активном ограничении:
# qdisc tbf 8001: root refcnt 2 rate 1Mbit burst 15200b lat 50ms
```

## Автоматическое снятие по таймауту

```bash
nohup bash -c "sleep 1200 && sudo tc qdisc del dev eth0 root 2>/dev/null && rm -f /tmp/tc-chaos.pid" \
    >/dev/null 2>&1 &
echo $! > /tmp/tc-chaos.pid
```

Процесс живёт независимо от SSH-сессии.

## Досрочное снятие (-D)

```bash
./07-net-bw.sh -D
```

Или вручную на хосте:
```bash
if [ -f /tmp/tc-chaos.pid ]; then
    kill $(cat /tmp/tc-chaos.pid) 2>/dev/null || true
    rm -f /tmp/tc-chaos.pid
fi
sudo tc qdisc del dev eth0 root
```

## Проверка статуса (-C)

```bash
./07-net-bw.sh -C
./07-net-bw.sh -C node-b.example.net
```

Вывод (пример):
```
node-a.example.net (eth0)  chaos: ACTIVE   [tbf] rate=1Mbit burst=15200b latency=50ms

RTT → 2001:db8::1 (node-b.example.net):19001  (10 проб)
   0.41 ms  |##
   0.38 ms  |##
   avg: 0.39 ms
```

RTT при TBF меняется мало — ограничение проявляется в пропускной способности (throughput),
а не в задержке. Для наблюдения эффекта используйте `iperf3` или метрики YDB.

## Команды для ручного применения

Команды для копирования на стенд. Параметры: полоса **1 мбит/с**, burst **15625 байт**, интерфейс **`eth0`**.
Адаптация: замените интерфейс, значение rate, пересчитайте burst (см. формулу выше).

Для нескольких нод — выполнить те же команды на каждом хосте.

### IPv6 и IPv4 (команды одинаковы)

TBF ограничивает исходящий трафик на уровне интерфейса без протокол-специфичных фильтров —
работает одинаково для IPv4 и IPv6.

**Применить (на каждом нужном хосте):**
```bash
sudo tc qdisc del dev eth0 root
sudo tc qdisc add dev eth0 root tbf rate 1mbit burst 15625 latency 50ms
```

**Проверить:**
```bash
tc qdisc show dev eth0
# Ожидаемый вывод:
# qdisc tbf 8001: root refcnt 2 rate 1Mbit burst 15200b lat 50ms
```

**Снять:**
```bash
sudo tc qdisc del dev eth0 root
```

**Примеры для других значений rate:**

| rate     | burst (авто)   | Команда                                                                   |
|----------|----------------|---------------------------------------------------------------------------|
| 1 мбит   | 15 625 б       | `tc qdisc add ... tbf rate 1mbit burst 15625 latency 50ms`               |
| 10 мбит  | 156 250 б      | `tc qdisc add ... tbf rate 10mbit burst 156250 latency 50ms`             |
| 100 мбит | 1 562 500 б    | `tc qdisc add ... tbf rate 100mbit burst 1562500 latency 50ms`           |
| 500 мбит | 7 812 500 б    | `tc qdisc add ... tbf rate 500mbit burst 7812500 latency 50ms`           |
| 2 гбит   | 31 250 000 б   | `tc qdisc add ... tbf rate 2gbit burst 31250000 latency 50ms`            |

Расчёт burst: `burst = rate_mbit * 15625` (минимум 1600).

> **`latency` ≠ добавленная задержка.** Параметр задаёт максимальное время ожидания пакета
> в очереди перед дропом (`queue_size = rate × latency`). Реальную задержку трафика не увеличивает.
> Обязателен в синтаксисе tc tbf — без него или без `limit` команда не выполнится.
